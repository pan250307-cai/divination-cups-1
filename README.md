# 掷圣杯游戏部署指南

本指南将帮助你将掷圣杯游戏部署到互联网上，让其他人可以访问和使用。

## 部署选项

### 1. GitHub Pages GitHub Pages（推荐，完全免费）

GitHub Pages 是一个免费的静态网站托管服务，非常适合部署简单的HTML游戏。

#### 步骤：

1. **创建GitHub账号**
   - 如果你还没有GitHub账号，请访问 [https://github.com/](https://github.com/) 注册一个免费账号。

2. **创建新仓库**
   - 登录后，点击右上角的 "+" 图标，选择 "New repository"。
   - 仓库名称建议使用 `divination-cups` 或其他你喜欢的名称。
   - 选择 "Public"（公开），勾选 "Add a README file"。
   - 点击 "Create repository"。

3. **上传游戏文件**
   - 进入新创建的仓库，点击 "Add file" -> "Upload files"。
   - 将 `index.html` 文件和所有相关资源（如图片等）拖拽到上传区域。
   - 滚动到底部，点击 "Commit changes"。

4. **启用GitHub Pages**
   - 在仓库页面，点击顶部的 "Settings" 选项卡。
   - 滚动到 "Pages" 部分。
   - 在 "Source" 下拉菜单中，选择 "main" 分支（或 "master" 分支）。
   - 在 "Folder" 下拉菜单中，选择 "/"（根目录）。
   - 点击 "Save"。
   - 稍等几分钟，页面会显示你的网站URL（通常是 `https://<你的用户名>.github.io/<仓库名>/`）。

5. **访问你的游戏**
   - 使用显示的URL访问你的掷圣杯游戏。

### 2. 使用 Netlify（推荐，免费计划可用）

Netlify 提供更强大的静态网站托管服务，支持自动部署和全球CDN。

#### 步骤：

1. **创建Netlify账号**
   - 访问 [https://www.netlify.com/](https://www.netlify.com/) 注册一个免费账号。

2. **创建新站点**
   - 登录后，点击 "New site from Git"。
   - 选择 "GitHub" 作为Git提供商（你需要授权Netlify访问你的GitHub账号）。
   - 选择你的游戏仓库。
   - 在 "Build command" 中留空（因为这是静态网站）。
   - 在 "Publish directory" 中输入 "/"（根目录）。
   - 点击 "Deploy site"。

3. **访问你的游戏**
   - Netlify会自动部署你的网站，并提供一个随机生成的URL。
   - 你可以在项目设置中自定义域名（需要购买域名）。

### 3. 使用 Vercel（推荐，免费计划可用）

Vercel 是另一个流行的静态网站托管平台，特别适合前端项目。

#### 步骤：

1. **创建Vercel账号**
   - 访问 [https://vercel.com/](https://vercel.com/) 注册一个免费账号。

2. **导入项目**
   - 登录后，点击 "New Project"。
   - 选择 "Import from Git Repository"。
   - 连接你的GitHub账号，选择游戏仓库。
   - 配置项目设置（通常默认设置即可）。
   - 点击 "Deploy"。

3. **访问你的游戏**
   - Vercel会自动部署你的网站，并提供一个URL（通常是 `<项目名>-<用户名>.vercel.app`）。

### 4. 使用传统Web服务器

如果你有自己的Web服务器或虚拟主机，可以通过FTP或文件管理器上传文件。

#### 步骤：

1. **准备文件**
   - 确保你有 `index.html` 文件和所有相关资源。

2. **连接到服务器**
   - 使用FTP客户端（如FileZilla）或你的主机提供商提供的文件管理器。
   - 连接到你的Web服务器。

3. **上传文件**
   - 将所有游戏文件上传到服务器的公共目录（通常是 `public_html`、`www` 或 `htdocs`）。

4. **访问你的游戏**
   - 使用你的域名或服务器IP地址访问游戏。

## 注意事项

1. **文件路径**：确保所有资源文件（如图片）的路径在HTML中是正确的。如果使用相对路径，确保文件结构正确。

2. **跨域问题**：如果游戏使用了外部资源，可能会遇到跨域问题。确保所有资源都来自允许跨域访问的服务器。

3. **性能优化**：对于更好的用户体验，可以考虑：
   - 压缩图片和JavaScript文件
   - 使用CDN加载第三方库
   - 启用Gzip压缩

4. **自定义域名**：大多数托管平台支持自定义域名，你可以从域名注册商（如Namecheap、GoDaddy等）购买域名，并按照平台指南配置。

## 故障排除

1. **网站无法访问**：
   - 检查文件是否正确上传
   - 确保index.html文件在根目录
   - 检查拼写错误和路径问题

2. **图片不显示**：
   - 检查图片路径是否正确
   - 确保图片文件已上传到服务器
   - 检查图片文件名的大小写（某些服务器区分大小写）

3. **JavaScript错误**：
   - 使用浏览器的开发者工具（按F12）检查控制台错误
   - 确保所有JavaScript代码正确无误

## 总结

GitHub Pages 是最简单和完全免费的选项，适合个人项目和小型应用。Netlify 和 Vercel 提供更多高级功能和更好的性能，也有免费计划可用。选择最适合你需求的部署方法，让更多人可以体验你的掷圣杯游戏！
