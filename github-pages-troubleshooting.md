# GitHub Pages 部署问题排查指南

如果你在使用GitHub Pages部署掷圣杯游戏后遇到显示问题？本指南将帮助你解决常见问题。

## 快速检查清单

### 1. 文件位置是否正确？
- ✅ `index.html` 必须在仓库根目录
- ❌ 不要放在子文件夹中

### 2. GitHub Pages 设置是否正确设置？
- 进入仓库 → Settings → → Pages
- ✅ Source 设置为正确分支（通常是 main）
- ✅ Folder 设置为 "/"（根目录）
- ✅ 看到绿色提示 "Your site is published at..."

### 3. URL 是否正确？
- 正确格式：`https://<你的用户名>.github.io/<仓库名>/`
- 检查拼写错误

## 常见问题和解决方案

### 问题1：显示404错误（页面未找到）

**解决方案：**
1. 确认 `index.html` 文件存在于仓库根目录
2. 检查GitHub Pages设置是否正确
3. 等待几分钟，部署可能需要时间
4. 清除浏览器缓存后重试

### 问题2：页面显示不正常（样式丢失、图片不显示）

**解决方案：**
1. **检查资源路径**
   - 使用相对路径，如 `./images/cup.png`
   - 避免使用绝对路径

2. **添加.nojekyll文件**
   - 在仓库根目录创建空文件 `.nojekyll`
   - 提交并推送这个文件到GitHub

3. **检查HTML中的引用**
   - 确保所有 `<link>`、`<script>` 和 `<img>` 标签路径正确

### 问题3：只显示README文件内容

**解决方案：**
1. 确保 `index.html` 文件存在于仓库根目录
2. 检查GitHub Pages设置中的"Source"是否正确
3. 确认没有其他文件（如index.md）覆盖了index.html

### 问题4：图片不显示

**解决方案：**
1. 检查图片路径是否正确
2. 确保图片文件已上传到GitHub仓库
3. 注意文件名的大小写（区分大小写）

## 逐步排查步骤

1. **检查文件结构**
   ```
   divination-cups/
   ├── index.html          # 主HTML文件（必须在根目录）
   ├── README.md           # 说明文件
   └── .nojekyll           # 禁用Jekyll的空文件（推荐添加）
   ```

2. **本地测试**
   - 在本地打开 `index.html` 文件，确保可以正常显示
   - 如果本地也有问题，说明代码本身有问题

3. **查看部署状态**
   - 访问仓库的 "Actions" 标签页，查看部署是否成功
   - 如果失败，查看错误日志获取更多信息

## 替代部署方案

如果GitHub Pages仍然无法正常工作，你可以尝试其他免费的静态网站托管服务：

1. **Netlify** - 提供简单的拖拽上传功能
2. **Vercel** - 适合前端项目的部署平台
3. **Firebase Hosting** - Google提供的托管服务

这些平台通常提供更简单的部署流程和更好的错误提示。
